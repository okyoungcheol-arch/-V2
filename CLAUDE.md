# 프로젝트명: 전세버스 통합 관리 ERP (Bus Management System)

## 1. 개발 환경 및 CLI 설정
- **Sub-Agent Model**: `CLAUDE_CODE_SUBAGENT_MODEL: "haiku"` (비용 효율적 하위 작업 처리)
- **Security Harness**: `AgentShield` 적용 (Safe-mode 기반 보안 강화)
- **Core Skills**: 
  - `tdd-workflow`: 모든 핵심 비즈니스 로직(배차, 정산)은 테스트 코드를 먼저 작성한다.
  - `strategic-compact`: 코드 밀도를 높이고, 중복 없는 전략적 아키텍처를 유지한다.

## 2. 데이터베이스 및 보안 (Supabase)
- **RBAC**: `직원_기사.역할`에 따른 엄격한 접근 제어.
- **RLS**: Row Level Security 활성화로 데이터 격리.
- **Auth**: 기사 로그인 보안 및 암호화 필수.

### DB 테이블 스키마 (주요 변경 컬럼)

| 테이블명 | 주요 컬럼 |
| :--- | :--- |
| **[직원_기사]** | 이름, 사번, 연락처, 역할, 면허번호, 상태, 운전가능(Y/N), 직원유형, 월급여, 로그인ID, 비밀번호, 로그인횟수, 최종로그인일시, 비고 |
| **[배차]** | 운행일자, 노선코드, 차량번호, 기사이름, 변경사유, 운행금액, 기사수당, 부가세포함여부, 등록일시, 상태, 비고 |
| **[운행정산]** | ~~삭제됨~~ |

## 3. 핵심 비즈니스 로직 (Logic Flow)
### A. 배차 및 상태 관리
- 현재일 기준 7일 전 [노선] 기반 [배차] 자동 생성.
- `운행일자 == 오늘` & `시간 >= 현재` 시 '운행완료' 자동 전환.

### B. 금융 처리
- [할부계약정보] 기반 [할부스케쥴] 자동 생성.
- [은행거래] 업로드 시 '적요' 분석을 통한 [금전출납부] 다중 분할 입력.

## 4. 디자인 시스템 (Design Identity)
- **Theme**: Primary `#1E40AF` (Deep Blue), 가독성 중심의 `Pretendard` 폰트.
- **Admin**: 사이드바 대시보드 구조 (Shadcn UI 활용).
- **User(기사)**: 모바일 최적화, 하단 탭바, 대형 버튼(44px+) 중심의 직관적 UI.

## 5. AgentShield 및 워크플로우 가이드
- **작업 시작 전**: `strategic-compact`를 통해 구현 전략을 요약 보고할 것.
- **코드 작성 시**: `tdd-workflow`에 따라 테스트 파일을 먼저 생성하고 통과 여부를 확인할 것.
- **보안 검토**: 모든 DB 쿼리와 API 호출은 AgentShield 가이드라인에 따라 잠재적 취약점을 점검할 것.

## 6. 메뉴의 순서 (`/URL`)
- 대시보드    (`/dashboard`)
- 배차관리    (`/dispatches`)
- 할부관리    (`/installments`)
- 은행거래    (`/bank`)
- 금전출납부  (`/ledger`)
- 노선관리    (`/routes`)
- 차량관리    (`/vehicles`)
- 직원관리    (`/employees`)
- 고객사      (`/clients`)

## 7. 메뉴의 사이드바
- `◀ / ▶` 버튼으로 `w-60 (펼침)` ↔ `w-16 (아이콘만)` 전환.
- 접힌 상태에서는 아이콘만 표시, `title` 속성으로 툴팁 제공.

## 8. 화면스크롤
- 전체화면이 아닌, 데이터 부분으로 스크롤 이동, 헤드는 틀고정하고, 정렬 기능 추가

## 9. 금액 부분 입력 방식
- 금액입력부분 숫자0이 표시되는데,표시안되게 해주고, 입력이 없으면 0로 저장 되게해줘, 금액이 입력되면 천단위 구분기호로 표시해주고, 저장시 숫자로 저장되게 해줘
